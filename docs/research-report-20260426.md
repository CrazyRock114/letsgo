> 日期：2026-04-26
> 研究范围：RAG 解说系统优化路径、KataGo 模型选型、教学体系对标

## 一、PAGE 数据集 — 职业对局权威数据源

### 1.1 数据集概况

[PAGE (The ProfessionAl Go annotation datasEt)](https://github.com/yifangao112/PAGE) 是目前围棋领域最权威的大规模标注数据集：

| 指标 | 数值 |
|------|------|
| 对局总数 | **98,525 局** |
| 职业棋手 | 2,007 位 |
| 时间跨度 | 1950–2021（70+ 年） |
| 数据大小 | ~59.6 GB |
| 许可证 | CC BY-NC-SA 4.0（非商业） |

### 1.2 数据内容

PAGE 包含每步棋的 KataGo 深度分析：
- **Ownership maps**（领地预测，28.5 GB）
- **Recommended moves**（推荐落点，5.49 GB）
- **In-game statistics**（胜率、目差、不确定性估计，1.26 GB）
- **Metadata**（棋手信息、赛事、日期，13.3 MB）

### 1.3 关键结论

**PAGE 不包含自然语言解说文本**。其"annotation"指的是统计/AI分析标注，而非人类撰写的"这步棋是好棋因为..."。

**对我们的意义**：
- 可以导入全部 98K 局职业对局，将 RAG 知识库从 26K 扩充到 **120K+ 位置**
- 可以利用 KataGo 的 ownership map 和推荐落点数据，走"可解释 AI"路线
- 不能直接用于训练 LLM 解说文本

## 二、行业平台对比 — 6 家主流围棋 AI 教学系统

### 2.1 综合对比表

| 维度 | 99围棋 | 弈客少儿 | 聂卫平道场 | 星阵围棋 | 围棋微学堂 | 书生·思客 |
|------|--------|---------|-----------|----------|-----------|-----------|
| **AI引擎** | 自研 | 自适应 | 自研 | 自研（神算子） | KataGo | 大模型 |
| **AI复盘** | 即时分析、无限次免费 | AI复盘+简单语言 | 对弈-复盘-精讲 | 胜率曲线+最佳走法 | 胜率曲线+关键失误 | **可解释推理** |
| **分级体系** | 无明确分级 | **知识地图** | 10级AI陪练 | **37个AI等级** | 全段位 | 多种风格 |
| **专项训练** | 无 | 无 | **2000+专项题库** | **9大专项特训** | 职业讲解 | 无 |
| **能力追踪** | 无 | **12项维度报表** | 学情报表 | 无 | 无 | 无 |
| **教学模式** | GPU实时分析 | 线上课堂 | OMO混合+AR 3D | 特训模式 | 双视角直播 | **自然语言解释** |
| **目标用户** | 4-15岁儿童 | 少儿学员 | 道场学员 | 全段位 | 全段位 | 全段位 |

### 2.2 各平台核心亮点

**99围棋** — 2024年重点更新 19 路 AI 分析，布设大量 GPU 服务器支持即时计算，无需等待。

**弈客少儿** — "知识地图"分级体系 + 12 项能力维度追踪（吃子、做眼、连接等），学情数据报表。

**聂卫平围棋道场** — "对弈-复盘-精讲"三维循环，OMO 混合式教学，AR 3D 棋盘，2000+ 专项题库。

**星阵围棋** — 37 个 AI 等级（从业余到超一流），9 大专项特训（布局、手筋、死活、中盘、官子等），大赛 AI 全自动解说直播。

**围棋微学堂** — 职业棋手 + AI 双视角直播解说，支持线下棋盘拍照识别转数字棋谱。

**书生·思客 InternThinker**（上海AI Lab）— **全球首个打破"黑盒"的围棋大模型**：能用自然语言解释每一步的推理逻辑和决策依据，具备多种解说风格（鼓励/毒舌等）。这是从"给出数据"到"解释原因"的质变。

### 2.3 2024-2025 行业趋势

1. **从"给出数据"到"解释原因"** — InternThinker 代表的方向
2. **实时性、免费化** — 99围棋的无限次免费即时分析
3. **OMO 智能课堂** — 线上线下融合的智能化教学
4. **能力维度追踪** — 不只看胜率，看具体弱项

## 三、KataGo 模型对比研究

### 3.1 官方模型架构

来源：[katagotraining.org/networks](https://katagotraining.org/networks/)

| 模型 | 参数量估计 | Elo 估计 | 用途 | 适用性评估 |
|------|-----------|---------|------|-----------|
| **b28c512nbt** | ~400MB | ~14,000+ | 当前最强主力 | 棋力最强但慢，适合分析 |
| **b18c384nbt** | ~180MB | ~13,600+ | 速度/棋力平衡 | **推荐** — 适合实时教学 |
| **b10c128** | ~40MB | 较低 | 老型号 | 太弱，不建议测试 |
| **rect15-b20c256**（当前用）| ~87MB | 未知 | 非官方定制模型 | 已验证稳定，但棋力上限未知 |

### 3.2 对比测试建议

**测试目标**：不是比"谁棋力强"，而是比"谁更适合做教学解说"

| 测试指标 | 说明 |
|---------|------|
| 胜率稳定性 | 同一局棋连续分析 3 次，胜率波动 < 2% 为优 |
| 目差一致性 | 不同模型对同一局棋的目差评估是否一致 |
| 推荐落点重合度 | top3 推荐中有几个重合 |
| 速度 | 500 visits 下的分析时间 |
| 内存占用 | 运行时 RAM 占用 |

**推荐对比组合**：rect15（当前）vs b18c384nbt（平衡型）vs b28c512nbt（最强型）

## 四、可解释 AI 路线 — InternThinker 启示

### 4.1 核心差异

| 维度 | 传统 AI（KataGo/AlphaGo） | 可解释 AI（InternThinker） |
|------|--------------------------|---------------------------|
| 输出 | 胜率 + 推荐落点 | 自然语言解释"为什么下这里" |
| 交互 | 单向数据展示 | 双向问答（可追问） |
| 风格 | 固定 | 多种风格（鼓励/毒舌/专业） |
| 教学价值 | 中（需用户自己理解） | 高（直接讲棋理） |

### 4.2 我们的可行路径

当前我们的系统已经接近可解释 AI，但缺少：
1. **Ownership map 可视化数据** — "这里黑方有 80% 的概率成地"
2. **Policy heatmap** — "AI 认为这 5 个点是最好的选择"
3. **变化图分析** — "如果白方不下这里，黑方下一步可以切断"

**实现方案**：
- 短期：在 prompt 中增加 KataGo 的 ownership map 统计（如"黑方在此处领地概率 75%"）
- 中期：前端增加领地预测可视化
- 长期：训练专用"解说模型"（类似 InternThinker）

## 五、教学体系对标分析

### 5.1 我们的现状 vs 行业标杆

| 模块 | 我们的现状 | 行业标杆 | 差距等级 |
|------|-----------|---------|---------|
| **分级体系** | easy/medium/hard（3级） | 星阵 37 级 / 聂道场 10 级 | 大 |
| **专项训练** | 无 | 聂道场 2000+ 题库 / 星阵 9 大专项 | 大 |
| **能力追踪** | 无 | 弈客少儿 12 项维度 | 大 |
| **AI复盘** | 胜率+目差+推荐 | 99围棋即时分析 / 星阵特训 | 中 |
| **AI对弈** | 有（3难度） | 全平台标配 | 小 |
| **课程内容** | 百科（简单） | 聂道场 OMO / 微学堂职业视频 | 大 |
| **RAG解说** | JSON事实+职业佐证 | InternThinker 可解释推理 | 中（有独特优势） |

### 5.2 我们的差异化优势

1. **幻觉率 0%** — JSON 事实骨架 + 结构化 prompt，这是行业首创
2. **RAG 职业佐证** — 能引用真实职业对局，增加说服力
3. **儿童友好** — 专门为儿童设计的解说风格（其他平台多为成人向）

### 5.3 需要补强的短板

1. **分级粒度太粗** — 3 个级别无法精准匹配儿童水平
2. **无专项训练** — 儿童吃子弱、做眼弱，无法针对性练习
3. **无能力追踪** — 家长看不到孩子的具体进步和薄弱点
4. **课程内容单薄** — 只有百科，无系统化课程

## 六、结论与建议

### 6.1 数据层策略

| 优先级 | 行动 | 预期效果 |
|--------|------|---------|
| P0 | 导入 PAGE 数据集 98K 局 | RAG 知识库 4 倍扩充 |
| P1 | ai-test 棋局自动入库 | 持续补充 9x9/13x13 真实对局 |
| P2 | OGS 大数据集筛选 | 补充网上娱乐对局 |

### 6.2 模型层策略

| 优先级 | 行动 | 预期效果 |
|--------|------|---------|
| P0 | KataGo 模型对比测试（rect15 vs b18 vs b28） | 选定最优教学模型 |
| P1 | 评估是否升级默认模型 | 提升分析质量 |

### 6.3 教学体系策略

| 阶段 | 目标 | 参考标杆 |
|------|------|---------|
| 短期（1-2周） | 分级体系 redesign（能力导向） | 弈客少儿知识地图 |
| 中期（1-2月） | 专项训练模块（吃子、做眼等） | 聂道场 2000+ 题库 |
| 长期（2-3月） | 系统化课程内容 + 能力追踪 | 星阵特训 + 弈客报表 |

### 6.4 解说质量策略

| 阶段 | 目标 | 参考标杆 |
|------|------|---------|
| 短期 | 优化 prompt，增加 ownership map 数据 | InternThinker |
| 中期 | A/B 测试框架，量化解说质量 | 星阵大赛解说 |
| 长期 | 训练专用"儿童围棋解说模型" | InternThinker |
